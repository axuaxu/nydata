    Good Morning everyone!

    I am currently creating a script that pulls multiple reports such as the current SEP date, Last boot time, inactive computers, ect, and the info is divided into different CSV files so that way they can have their own sets of info. Now I would like to export these CSV's into a HTML format so I can have it automatically sent out in an email every morning. However, whenever I run "import-csv .\file.csv | convertTo-HTML | Out-File .\file.html" I do run some additional -Head and -Body commands just to pretty it up, however whenever I run the import command for the second csv it overwrites the first, is there a way so it will just combine? (I know it can be accomplished with VBS I am just trying to stick to powershell at the moment).

    Thank you for your time!

    You can use the -Append switch of Out-File to have it add to the HTML file.

    Powershell
    Import-Csv .\file.csv | ConvertTo-Html | Out-File .\file.html -Append

    Another option, depending on your requirements, would be to use the -Fragment switch of ConvertTo-Html and build the entire HTML body before exporting it to a file.
    Powershell
    $Files = Get-ChildItem -Path 'C:\CSVDir' -Include *.csv

$HTML = foreach( $File in $Files ){
    Import-Csv -Path $File.FullName |
        ConvertTo-Html -Fragment
    }

$HTML |
    Out-File -FilePath 'C:\Report.html'

    Good morning!

    Are you saying it is overwriting the html file?

    You may need to have them exported as two separate files first and then combine them...

    I would combine the CSV's first and then export the final one to HTML. You can achieve that with something like:

    Powershell
    $csv1 = @(gc File1.csv)
$csv2 = @(gc File2.csv)
$csv3 = @()
for ($i=0; $i -lt $csv1.Count; $i++) {
    $csv3 += $csv1[$i] + ',' + $csv2[$i]
    }

$csv3
    Powershell
    $csv4 | convertto-HTML | Out-File file.html
